## REQUISITOS FUNCIONAIS (por ordem de prioridade)

### 1. AUTENTICAÇÃO E PERFIS
- Login/registro com email/senha via Supabase Auth
- Perfil com username, avatar e reputation_scoreore (0-100)
- Middleware de proteção de rotas

### 2. MARKETPLACE DE CARTAS
- Listagem paginada com filtros (nome, set, condition, preço)
- Cards mostram: imagem, nome, preço, condição, reputação do vendedor
- Apenas cartas com status='available' e verified_by_admin=true aparecem
- Upload de cartas com múltiplas fotos (frente/verso/detalhes)

### 3. SISTEMA DE TRADES COM CHAT EM TEMPO REAL
- Iniciar trade a partir de uma carta (bloqueia carta para outros)
- Chat em tempo real usando Supabase Realtime subscriptions
- Tipos de mensagem: texto, oferta de preço, sistema (notificações automáticas)
- Status da trade: negotiating → payment_pending → payment_held → shipped → delivered → completed

### 4. SISTEMA DE PAGAMENTO INTERNO (ANTI-GOLPE)
- Wallet com saldo em centavos (evitar float)
- Escrow (garantia): dinheiro fica retido até confirmação de recebimento
- Edge Functions para processamento seguro (nunca no client)
- Transações: deposit, withdrawal, payment, receipt, escrow_hold, escrow_release

### 5. PROTEÇÕES ANTI-GOLPE
- Reputação mínima para vender (>50)
- Prazos automáticos: 24h para pagar, 72h para enviar, 72h para confirmar recebimento
- Sistema de disputa com status 'disputed'
- Verificação manual de cartas por admin (verified_by_admin)

EDGE FUNCTIONS (implementar todas)
create-trade: Valida carta disponível, reputação do vendedor, cria trade, bloqueia carta
process-payment: Verifica saldo, move para escrow, atualiza status
release-payment: Libera para vendedor ou abre disputa
verify-card: (Futuro) Análise de imagem por IA ou manual
